Fichier de configuration:
	le fichier de configurations est fait par les dev du projets il peut donc contenir 
	les configurations des differentes features du serveur web

	Le fichier de configurations se repartit en plusieurs "context" et chaque context a son contexte

	On peut y trouver:
		le context main qui est en haut du fichier de configuration: (je sais pas a quoi il sert mais il est la)

		le context event
		{
			- le choix sur la gestion des connexions (select poll epolll kqueu):
				mais dans notre cas on utilisera epoll.

			- le nombre de connexions maximum
		}

		le context http qui est considere comme "frere" au context event la consideration de "frere"
		me semble etre figure, ce qu'il faut retenir c'est qu'ils sont cote a cote dans le fichier de configuration
		en dessous de main context

		le context http
		{
			- certaine directive que le serveur est susceptible de rencontrer
				- l'emplacement des erreur log (acces log et error log)
				- sendfile on | off -> nous ne l'utilisons pas, si sendfile est en on alors il faut passer par 
					l'apel systeme sendfile (fonction non autorise)
		}

		le context server qui est imbriquer dans le context http, il peut y avoir autant de context serveur qu'on veut
		chaque context serveur servira a repondre a un type de serveur demande par le client
